# 끝말 잇기 V1 — 탭→드롭존 채우기 + 더미 멀티라운드 설계

작성일: 2026-06-26

## 목적

현재 디자인 목업(`frontend/v1/mockup/`)의 중앙 작은 `?` 원을 **큰 드롭존 빈 카드**로
바꾸고, **보기 카드를 탭하면 그 카드가 드롭존으로 날아가 채워지는** 상호작용을 추가한다.
채워지면 미리 짜둔 더미 시나리오의 다음 라운드로 진행해, 실제 플레이 흐름(연결되는
느낌·진행감)을 검증한다.

이 작업은 **목업 안에서 완결**한다. 엔진 연결·진짜 프론트와의 통합/동기화는 **다음 단계**이며
이 스펙의 범위 밖이다.

## 버전 구분 (확정)

- **v1 (이번 작업):** 보기 카드 **탭** → 카드가 드롭존으로 자동으로 날아가 채워짐.
- **v2 (이후):** 카드를 **직접 드래그**해 드롭존에 끌어다 놓기.
- 두 버전은 **같은 "큰 드롭존" 무대를 공유**하고 입력 방식만 다르다. v1을 먼저 만든다.

## 범위

### 포함 (이번 v1)
- 작은 원형 `?`(현재 `.slot`) → **보기 카드와 같은 크기의 큰 빈 드롭존**으로 교체.
- 보기 카드 탭 → 드롭존으로 날아가 안착하는 애니메이션 + "연결됨" 피드백.
- 미리 짜둔 **6라운드 더미 시나리오**로 라운드 진행(라운드 갱신 + 진행도 점 채움).
- 6라운드 후 **간단한 완주 상태**.
- 단일 자기완결 `mockup.html` 재생성(`build.py` 확장) → 같은 아티팩트 URL로 재배포.

### 제외 (명시적 비범위)
- 드래그앤드롭(= v2).
- 엔진(`kkutu-engine.js`) 연결 / 진짜 프론트(`frontend/`)와의 동기화·통합.
- 전체 체인 회고 화면(지나온 단어 전체 표시) — HANDOFF의 별도 *완주 화면*에서 다룰 예정.
- 끝말잇기 규칙·금칙어·재사용 검증(엔진 몫). v1 더미에선 **오답도 그냥 진행**한다.
- 음성(마이크) 입력 동작. 마이크 카드는 현재처럼 표시만 유지하고 탭해도 아무 동작 없음(no-op).

## 확정된 단순화 결정

1. **완주는 간단 상태로만** — "다 이었어요!" 수준의 마무리. 전체 회고 화면은 만들지 않는다.
2. **오답도 진행** — 어느 보기를 탭하든 그 단어가 드롭존에 채워지고 다음 라운드로 간다.
   규칙 검증은 엔진 도입 때 붙인다.

## 아키텍처 / 구현 방식

기존 목업 빌드 패턴을 그대로 확장한다.

| 파일 | 변경 |
|---|---|
| `frontend/v1/mockup/assets.json` | 더미 체인에 필요한 픽토그램 base64 추가(기존 8개 → 총 19개). 폰트는 그대로. |
| `frontend/v1/mockup/build.py` | `SCENARIO` 배열을 JS로 임베드, 첫 라운드 렌더, 탭→채움→다음라운드→진행도 JS 추가. |
| `frontend/v1/mockup/mockup.html` | `build.py` 산출물(자기완결). 직접 편집 금지, 항상 빌드로 생성. |

- 산출물은 외부 요청이 없는 단일 HTML이라 아티팩트 CSP를 만족한다.
- 픽토그램 원본은 `frontend/v1/img/<pic>.png`, 단어 메타는 `frontend/v1/tg_words.json`에 있다.
  base64 인라인 보조 스크립트가 필요하면 build 단계에서 png를 읽어 data URI로 변환한다.

### 데이터 모양 (엔진 통합 대비)

`SCENARIO` 한 라운드의 형태를 **나중에 엔진 `toView(state)`가 내놓는 모양과 정렬**해 둔다.
엔진 통합 시 더미 배열을 엔진 출력으로 갈아끼우는 것만으로 끝나게 하기 위함이다.

```js
const SCENARIO = [
  { now:"간호사", tail:"사", options:["사과","사탕","사자"],   answer:"사과" },
  { now:"사과",   tail:"과", options:["과자","과일","과학"],   answer:"과자" },
  { now:"과자",   tail:"자", options:["자전거","자동차","자두"], answer:"자전거" },
  { now:"자전거", tail:"거", options:["거북이","거위","거실"],   answer:"거북이" },
  { now:"거북이", tail:"이", options:["이야기","이불","이마"],   answer:"이야기" },
  { now:"이야기", tail:"기", options:["기차","기저귀","기린"],   answer:"기차" },
];
// now: "방금 만든 말"(직전 단어). tail: 공유 글자(now의 끝). options: 보기 3개. answer: 체인을 잇는 정답.
// 이미지는 word→data URI 매핑(assets)에서 조회.
```

## 더미 데이터 (자동 큐레이션 완료·검증됨)

체인: **간호사 → 사과 → 과자 → 자전거 → 거북이 → 이야기 → 기차** (6라운드)

| R | 방금 만든 말 | 공유 글자 | 보기 3개 (정답 굵게) |
|---|---|---|---|
| 1 | 간호사 | 사 | **사과** · 사탕 · 사자 |
| 2 | 사과 | 과 | **과자** · 과일 · 과학 |
| 3 | 과자 | 자 | **자전거** · 자동차 · 자두 |
| 4 | 자전거 | 거 | **거북이** · 거위 · 거실 |
| 5 | 거북이 | 이 | **이야기** · 이불 · 이마 |
| 6 | 이야기 | 기 | **기차** · 기저귀 · 기린 |

- 모든 단어(방금 만든 말 + 보기 19개)는 `tg_words.json` 출처이며 픽토그램 파일 존재 확인됨.
- 진행도 점: `TOTAL = 6`. 라운드 진행에 따라 현재◎ → 완료● 로 갱신.

## 화면 / 레이아웃 변경

현재 구도(상단바 · 히어로 · 보기행 3단)는 유지. 변경점:

- **드롭존(`.slot` 대체):** 작은 원형(18cqmin) → 보기 카드(`.opt`, 22cqw)와 **같은 크기의 큰 빈 카드**.
  앰버 점선 테두리 + `couple-soft` 배경 + breathe 애니메이션. 날아온 카드가 1:1로 안착하도록 크기 정렬.
- **구도 유지:** `[방금 만든 말 카드] ─[공유 글자 토큰]─ [큰 빈 드롭존]`.
- 색·폰트 토큰은 현재 `:root` 그대로 사용(앰버=연결, 그린=진행).

## 상호작용 루프 (v1)

```
보기 카드 탭
  → 카드가 드롭존 위치로 날아가 안착 (transform: translate + scale, ~0.35s)
  → "연결됨" 피드백: 드롭존이 정답 카드 모습으로 굳고 앰버 연결선이 이어짐 (~0.5s)
  → SCENARIO 다음 라운드로 갱신: 방금 만든 말 = 방금 채운 단어, 새 빈 드롭존, 새 보기 3개
  → 진행도 점 1개 채움 (현재◎ → 완료●)
  → 반복 (총 6라운드)
마지막 라운드 후
  → 간단한 완주 상태: "다 이었어요!" + 마스코트/체크 정도. 전체 회고 화면은 만들지 않음.
```

- **한 번에 활성 = 보기 3개뿐.** 드롭존·연결선·진행도는 표시 전용.
- 애니메이션 진행 중에는 보기 카드 재탭 방지(중복 입력 잠금).
- `prefers-reduced-motion: reduce`이면 날아가는 애니메이션 생략하고 즉시 안착·갱신.

## 접근성 / 품질 바닥선

- 보기 카드에 **`:active`(눌림) 상태 추가** — 태블릿 터치 피드백(기존 `:hover`만으로는 터치에서 반응 없음).
- `:focus-visible` 유지, `aria-label` 갱신(진행도 "n / 6", 드롭존 상태).
- 흐린 캡션 대비는 이번 범위에선 유지(별도 개선 항목). 필요 시 후속.

## 검증 방법

- 로컬: `python3 frontend/v1/mockup/build.py` → `mockup.html`을 브라우저(또는 로컬 서버)로 열어
  6라운드 진행·완주·진행도·reduced-motion 동작 확인.
- 아티팩트: `Artifact` 도구로 `mockup.html` 배포 시 기존 URL을 `url`로 넘겨 같은 곳에 덮어쓰기.

## 미해결 / 다음 단계

- v2: 드래그앤드롭 입력.
- 엔진 연결: `SCENARIO`를 엔진 `toView` 출력으로 교체 + 단어 데이터/엔진 JS 인라인.
- 진짜 프론트(`frontend/`)와 목업 시각 자산 수렴(하나의 코드베이스로 통합).
