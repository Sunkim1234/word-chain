# 끝말잇기 V1 — 드래그 스냅 + TTS 상호작용 설계

날짜: 2026-06-27
대상: `frontend/v1/mockup/mockup.html` (자체 완결 목업, 외부 라이브러리 없음)
관련: [2026-06-24 V1 프로토타입 설계](2026-06-24-wordchain-prototype-figma-make-design.md)

## 배경

현재 보기 카드(`.opt`)를 **탭하면** 카드가 드롭존 중심으로 `translate+scale`로 날아가고
400ms 뒤 `commit()`이 엔진을 진행시킨다 (`onPick()`).

이 탭-자동이동 방식을 **직접 드래그 + 근접 스냅** 방식으로 바꾸고,
탭/롱프레스는 단어를 **TTS로 읽어주는** 동작으로 재배치한다.

대상 사용자가 어린이(끝말잇기 학습)이므로 **관대함**이 최우선 원칙이다.

## 목표

1. 보기 카드를 손가락/마우스로 끌어다 빈 칸에 놓아 정답을 제출한다.
2. 카드가 빈 칸에 **살짝만 걸쳐도** 판정이 성공(armed)한다.
3. armed 상태에서 **손을 떼야** 확정된다 (스쳐 지나가도 실수 확정 없음).
4. armed 시 테두리가 빛나고 별 파티클이 반짝이는 시그니처 연출.
5. 카드를 탭하거나 꾹 누르면 단어를 한국어 TTS로 읽어준다.

## 비목표 (YAGNI)

- 탭으로 정답 확정 (제거 — 탭은 이제 "듣기").
- 틀린 카드 튕겨내기: 엔진이 보기를 **전부 정답**으로 보장하므로 불필요.
- 직전 단어 카드(`.now__card`) TTS: 이번 범위 제외.
- 외부 애니메이션/TTS 라이브러리: 자체 완결 유지.

## 결정된 상호작용 모델

### 입력

- **Pointer Events** (`pointerdown` / `pointermove` / `pointerup`) + `setPointerCapture`로
  터치·마우스를 단일 코드 경로로 처리.
- 기존 탭 핸들러 `onPick()` 제거.

### 제스처 구분 (option 카드 1개에서)

| 제스처 | 조건 | 동작 |
|---|---|---|
| 탭 | `pointerdown` 후 임계치 미만 이동, 빠르게 뗌 | 단어 **TTS 읽기** |
| 롱프레스 | 가만히 누른 채 ~400ms 경과 | 단어 **TTS 읽기** (누른 상태에서 즉시) |
| 드래그 | `pointerdown` 후 **8px 초과 이동** | 정답 놓기 모드로 전환, 읽기 타이머 취소 |

요약: **탭/꾹누르기 = 듣기, 끌기 = 답하기.**

### 근접 판정 — "살짝 걸치기만 해도"

- **면적 vs 면적**(사각형 교차)으로 판정. 점+반경 방식 아님.
- 끌고 있는 **카드의 실제 사각형**이 빈 칸의 **실제 사각형**과 **1px라도 겹치면 → armed**.
- 칸을 인위적으로 부풀리지 않는다. 카드가 크기 때문에 칸 테두리에 살짝만 닿아도 충분히 관대.
- 보이는 것(카드가 칸에 닿음)과 판정이 정확히 일치.

### 상태 흐름

| 상태 | 트리거 | 결과 |
|---|---|---|
| `idle` | 기본 | 점선 오렌지 칸 + "?" |
| `dragging` | 카드 누르고 8px 초과 이동 | 카드가 포인터 따라옴, 살짝 떠오름 |
| `armed` | 카드 사각형이 칸 사각형과 교차 | 칸 빛남 + 별 파티클 + 고스트 미리보기 |
| `commit` | armed 상태에서 손 뗌 | 칸 중앙으로 스냅 정렬 → 초록 테두리 → 엔진 진행 |
| `springback` | armed 아닐 때 손 뗌 | 카드 원래 자리로 탄력 복귀 |

확정 시 기존 `commit()` / `advance()` 엔진 흐름 재사용:
카드를 드롭존에 채우고 `engine.answer(word)` 호출 → 480ms 뒤 다음 라운드.

## 시각 · 모션

### 카드 (dragging)
- 포인터 따라 이동, `scale 1.05` + 그림자 강화 + 미세 기울기(~2°), `z-index` 최상단.
- 나머지 보기 카드 · 말하기 버튼은 `opacity .4`로 차분히 후퇴.

### 빈 칸 (armed) — 시그니처 순간
- 점선 → 실선 오렌지(`--couple`), `box-shadow` 글로우가 번지며 1회 맥동(pulse).
- **별 파티클**: armed 진입 순간 칸 둘레로 별(✦) 4~6개가 톡 터지고,
  머무는 동안 은은히 반짝임(scale + opacity 트위클, 시차 stagger).
- 색은 기존 팔레트만 사용: 카드 테두리 골드 `#F1B53C` + couple-orange `#EE9740`.
- 구현: DOM 별 스팬 + CSS 키프레임. 외부 라이브러리 없음.

### 확정 (commit)
- 손 떼면 카드가 칸 중앙으로 스냅 정렬(`cubic-bezier` 오버슈트).
- 테두리 초록(`--success`) — 기존 `.drop--filled` 스타일 재사용.
- 별이 한 번 더 크게 튀며 자축 후 사라짐.

### 복귀 (springback)
- 칸에 안 겹친 채 떼면 원래 슬롯으로 탄력 복귀.

### TTS 시각 신호
- 읽는 동안 카드에 절제된 신호 1회(살짝 커졌다 복귀 또는 스피커 펄스).

## TTS

- Web Speech `speechSynthesis` + `SpeechSynthesisUtterance(word)`, `lang = 'ko-KR'`.
- 한국어 음성이 있으면 우선 선택.
- 새 단어 읽기 전 `speechSynthesis.cancel()`로 진행 중 음성 중단(겹침 방지).
- 사용자 제스처(탭) 안에서 호출 → 자동재생 정책 통과.
- **best-effort**: 브라우저/OS에 한국어 음성이 없으면 읽히지 않을 수 있음(목업 한계).

## 접근성 / 품질 기준

- `prefers-reduced-motion`: 떠오름 · 기울기 · 바운스 · 별 파티클 · 맥동 제거.
  armed는 **정적 글로우**만, 확정은 즉시 스냅. (TTS는 모션과 무관, 시각 신호만 생략.)
- 키보드: 카드 포커스 후 Enter/Space로 확정(드래그 불가 사용자용 조용한 대안).
- 포커스 표시 유지(`:focus-visible`).
- 모바일 비율(기존 device 토글)에서 동작.

## 구현 범위

- **`mockup.html`은 `build.py`가 생성하는 산출물.** CSS·JS·HTML은 `build.py` 안의
  f-string으로 들어있으므로 **모든 수정은 `build.py`에서** 하고 재실행해 `mockup.html`을 재생성한다.
  (`mockup.html`을 직접 편집하지 않는다.)
- 수정 대상: `build.py`의 두 번째 `<script>` 블록(`onPick` / `fillDrop` / `commit` 등, 라인 235~)
  과 관련 CSS(`.opt` / `.drop` 등, 라인 143~). f-string이므로 JS의 `{` `}`는 `{{` `}}`로 이스케이프.
- 재빌드 후 Artifact로 배포해 동작 확인(HANDOFF.md 참고).
- 엔진(`KkutuEngine`)·라운드 렌더링·진행 표시·종료 화면 로직은 변경 없음.
