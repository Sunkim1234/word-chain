# 특수교육용 끝말잇기 프로토타입 — Figma Make 초안 설계

작성일: 2026-06-24

## 목적

공통 디자인 키트 도착 전, **구조·레이아웃·핵심 흐름 검증용 저충실도(low-fidelity)
인터랙티브 프로토타입**을 Figma Make로 생성한다. 비주얼 완성도가 아니라 "무엇이 / 어디에 /
어떤 순서로 배치되고, 어떻게 진행되는가"를 검증하는 것이 목표.

## 충실도 범위

이번 단계에서 잡는 것:
- 레이아웃 / 요소 배치 / 계층
- 폰트·터치 영역의 *상대적* 크기감 (정확한 px 아님, "크게/작게" 비율)
- 2개 화면 + 화면 전환 흐름
- 지정한 핵심 기능만

이번 단계에서 잡지 않는 것 (= 공통 디자인 키트 영역):
- 최종 컬러 팔레트
- 브랜드 폰트 / 아이콘 세트
- 버튼·카드의 최종 스타일링
- 그림자 / 애니메이션 / 마이크로인터랙션

스타일은 **중립 회색조 기반 + 강조용 색 1~2개만 기능적으로 사용**(현재 단어, 활성 버튼,
정답 피드백 강조). 브랜드 팔레트 / 완성된 컬러 스킴은 정하지 않는다 — 색만 갈아끼우면
키트로 교체 가능한 수준 유지.

## 특수교육 공통 원칙

- 한 화면 = 한 동작 (인지 부하 최소화)
- 항상 "지금 무엇을 해야 하는지"가 명확히 보이게
- 큰 글씨, 큰 터치 영역
- 시각적 노이즈 최소화

## 형태

가로형(태블릿) 단일 비율.

## 화면 ① 게임 플레이

요소:
- 상단 헤더: 뒤로가기 아이콘 + 제목 "끝말 잇기"
- 단어 체인: 이어진 단어가 가로로 표시 (예: 사과 → 과자 → 자동차 → **차표**),
  현재 단어 강조 + 부엉이 마스코트
- 안내 문구: "내 차례! '○'로 시작하는 말은?"
- 선택지 단어 버튼 3개
- 음성인식 버튼 1개

활성(클릭 가능) 요소: **선택지 3개 + 음성인식 버튼**, 이 4개뿐. 나머지는 표시 전용.

인터랙션 루프:
1. 선택지 버튼 클릭
2. 선택한 단어가 체인 맨 끝에 추가됨
3. 컴퓨터가 다음 시작 글자를 제시하고 선택지 3개가 갱신됨
4. 위를 **5라운드** 반복
5. 결과창으로 전환

비고: 선택지·다음 단어 데이터는 미리 짜둔 더미 시나리오로 동작. 실제 끝말잇기 로직 /
금칙어·재사용 검증은 백엔드 몫이며 이번 프로토타입 범위 밖.

## 화면 ② 결과창 (활동 마무리)

요소:
- 라운드 수 + 칭찬 메시지 (예: "5단어 완성! 잘했어요")
- 이어진 단어 체인 전체 시각화
- 사용한 단어 목록
- 처음 만난(새로 배운) 단어 따로 모아 표시

미포함: 다시하기 / 홈 버튼 (지정하지 않음).

## 명시적 비범위 (넣지 말 것)

- 지정하지 않은 임의 기능 일체
- 설정 / 점수판 / 타이머 / 로그인 등 기획에 없는 화면·기능
