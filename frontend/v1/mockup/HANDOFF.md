## 이 폴더 구성

| 파일 | 내용 |
|---|---|
| `mockup.html` | 현재 배포된 자기완결 목업(폰트·그림 base64 + 시나리오·인터랙션 JS 내장). **직접 편집 금지** — 항상 build.py로 생성. |
| `build.py` | `mockup.html`을 재생성. **디자인·시나리오·인터랙션 수정은 전부 여기서.** `python3 build.py`. 그림은 `assets.json`에서, **폰트는 `../font/`의 TTF를 그때그때 서브셋**해 인라인. |

## v1 인터랙션 (build.py 안에 구현됨)

- **시나리오:** `SCENARIO` 배열(라운드별 `now`/`tail`/`options`/`answer`). 한 라운드 모양을
  **엔진 `toView` 출력과 정렬**해 둠 → 엔진 통합 때 이 배열만 엔진 출력으로 교체.
- **탭 동작:** 보기 카드 클릭 → 원본 카드를 transform으로 드롭존까지 이동·스케일(~0.38s) →
  드롭존을 정답 카드로 채움(초록 테두리, pop) → ~0.48s 후 다음 라운드 렌더 + 진행도 점 채움.
- **reduced-motion:** 날아가는 애니메이션 생략, 즉시 채움.
- **오답 처리 없음:** 어느 보기를 눌러도 진행. 다음 "방금 만든 말"은 정해둔 정답 체인을 따라감(스펙대로).
- **마이크 카드:** 표시만, 탭해도 no-op.

간호사 → 사과 → 과자 → 자전거 → 거북이 → 이야기 → 기차 (6라운드)

## 확정된 디자인 결정 (V1)

- **대상:** 특수교육 아동 / **태블릿 가로**. **메인 기기 = 갤럭시 탭 A11+ (1920×1200, 16:10)**. 4:3·16:9도 안 깨지게.
- **톤:** 차분·저자극. 흰 카드 + 파스텔 배경(하늘→크림). **강조색은 의미 있는 곳에만** —
  amber `#EE9740`=연결 글자(끝말)/빈 드롭존, green `#54B98A`=진행/채워진 카드/완주.
- **폰트:** Jua(둥근 디스플레이=단어·프롬프트), Gowun Dodum(UI). 원본 TTF는 `frontend/v1/font/`(OFL 1.1, `OFL.txt` 동봉).
  build 때 **쓰는 글자만 서브셋**해 woff2로 인라인.
- **카드 색 규율(중요):** 색은 *의미*에만. amber=연결/끝말/빈 드롭존, green=진행/채워짐/완주, blue=음성(마이크).
  보기 카드는 "고를 것"이라 **무채에 가까운 웜그레이 테두리(`#DAD3C8`)** — 평소엔 조용하고, 골라서 green으로 변할 때만 색이 등장.
- **레이아웃(플레이 화면):**
  - 상단: 브랜드(좌) · **점-대시 진행도(중앙, done●/현재◎/남은—)** · 다시듣기 🔊(우).
  - 중앙(히어로): `[이전 단어 카드] ─(공유 글자)─ [큰 빈 드롭존]`. **드롭존이 주인공**(보기 카드와 같은 크기). 이전 단어 카드도 같은 크기.
  - 하단: **보기 카드 3개(웜그레이 테두리) + 슬림 마이크 카드(블루)** = 4칸. 보기 카드에 `:active` 눌림 피드백 있음.
- **반응형:** 컨테이너 기준 단위(`cqmin/cqw/cqh`)로 스케일, 스크롤 없이 한 화면.

## 새 단어 그림 추가하는 법

`build.py`의 `SCENARIO`에 새 단어를 쓰려면 그 단어가 `assets.json["assets"]`에 있어야 함.
없으면: `frontend/v1/tg_words.json`에서 단어의 `pic` id를 찾아 `frontend/v1/img/<pic>.png`를
base64로 인코딩해 `assets.json`에 `{"단어": {"pic":id, "uri":"data:image/png;base64,..."}}` 추가.
(이전에 14개를 그렇게 병합함 — 같은 패턴.)
