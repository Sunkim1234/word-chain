## 2026-06-27 — 드래그+스냅 · TTS · 컨베이어 전환

- **탭→드래그 전환**: 카드를 끌어 ?칸에 1px라도 겹치면 armed(테두리 글로우+별 파티클), 손 떼면 스냅 확정.
  탭/꾹누르기 = 단어 TTS(ko-KR). 누르면 살짝 떠올라 멈춤(드래그 회전 없음).
- **TTS 흐름**: 컴퓨터 차례엔 메인카드 팝+단어 읽고 0.3초 뒤 프롬프트 읽기. 정답 후 "직전단어→연결글자→내답"
  순으로 낭독+팝(연결감). 말풍선 🔊 버튼 = 프롬프트 다시 읽기. 파티클 끝난 뒤(~0.8s) 낭독 시작.
- **2단계 컨베이어 전환**: 정답 후 [메인 왼쪽 퇴장 → 답이 메인으로 → 컴퓨터가 ?칸에 새 단어 → 그게 메인으로].
  항상 메인1+?칸1만 보임. 위치는 `.laneCard` transform, 팝은 inner 카드 — 분리해 충돌 방지.
- **수정은 build.py에서만** → `python3 build.py`로 mockup.html 생성(직접 편집 금지). f-string이라 JS중괄호 `{{ }}`.
  검증: 빌드 + 2번째 `<script>` 추출해 `node --check` + 아티팩트 육안.
- **v1 / v2 = 독립 버전(상하 관계 아님)**: 라운드 전환 방식만 다른 **나란히 유지되는 두 변종**이다.
  v1 = **2단계 컨베이어** 슬라이드 전환, v2(`frontend/v2/mockup/`) = **즉시 교체(jump-cut)** 전환.

  올림 — `frontend/v1/mockup/mockup.html`(컨베이어) + `frontend/v2/mockup/mockup.html`(즉시 교체).
  둘 다 자체완결(폰트·이미지·엔진 인라인) → 더블클릭만으로 동작.

## v1 인터랙션 (build.py 안에 구현됨 — 엔진 구동)

- **상태 소유 = 엔진.** `new KkutuEngine(WORDS, {targetAnswers:5})` → `state = engine.start()`.
  `renderRound()`가 `state.lastWord`(now 카드)·`state.currentSyllable`(공유 글자/프롬프트)·
  `state.options`(보기 2~3개)·`state.childAnswers`(진행도)를 화면에 그린다. (더미 SCENARIO 없음.)
- **탭 동작:** 보기 카드 클릭 → `onPick` → 원본 카드가 transform으로 드롭존까지 이동·스케일(~0.38s,
  곁다리 연출) → **보장된 `setTimeout(commit, 400)`** → `commit`이 드롭존을 채우고 `engine.answer(word)`
  (아이+AI 한 수) → `setTimeout(advance, 480)` → `advance`가 다음 라운드 렌더 또는 완주.
  ⚠️ **진행을 `transitionend`에 의존시키면 안 됨**(이벤트 누락 시 멈춤 — 이번 세션 버그). 타이머로 보장.
- **reduced-motion:** 날아가는 애니메이션 생략, 즉시 `commit`.
- **오답 없음:** 엔진이 주는 보기는 전부 "끝까지 안 막히는 정답"이라 어느 걸 눌러도 안전.
- **마이크 카드:** 표시만, 탭해도 no-op.
- **RNG:** 없음(`Math.random`) → 매판 다른 게임. 같은 판 반복 원하면 `rng:시드함수` 주입.

## 새 단어 그림 추가하는 법

`build.py`의 `SCENARIO`에 새 단어를 쓰려면 그 단어가 `assets.json["assets"]`에 있어야 함.
없으면: `frontend/v1/tg_words.json`에서 단어의 `pic` id를 찾아 `frontend/v1/img/<pic>.png`를
base64로 인코딩해 `assets.json`에 `{"단어": {"pic":id, "uri":"data:image/png;base64,..."}}` 추가.
(이전에 14개를 그렇게 병합함 — 같은 패턴.)
